# 03 · 单层旋转（F 操作）

## 需求
- 在整体魔方上实现魔方记号意义下的 **F 操作**：把"正对相机的那一面"连同其相邻面上属于该层的一圈块一起旋转 ±72°。
- 页面上提供一个 **F 按钮** 触发该操作。
- 页面上提供一个 **朝向提示**，实时显示当前正前方面的颜色。
- 朝向跟随相机：相机转到哪，"前面"就是哪。

## 设计

### 渲染单元从"面"改为"块（Piece）"

当前代码按 12 个面分别渲染（每个面内部画 1 中心 + 5 棱片 + 5 角片）。这种做法下，相邻面共享的同一根棱在内存里是两个独立的 mesh，无法表达"块在转动后跨面交换"。

改为以"块"为单位渲染，魔方共 62 块：

| 类型 | 数量 | 携带彩色片 | 几何位置 |
|---|---|---|---|
| 中心块 (center) | 12 | 1 | 12 个面心 |
| 棱块 (edge) | 30 | 2 | 30 条棱中点 |
| 角块 (corner) | 20 | 3 | 20 个十二面体顶点 |

### 槽位（Slot）

每个块"当前所占的位置"称为一个槽位。槽位是固定的几何坐标 + 一组朝向基（局部 u/v/n），共 62 个。每个槽位预先记录：
- 它的中心坐标；
- 它"接触哪几个面"（中心 1 个，棱 2 个，角 3 个）；
- 它在每个所属面里的"局部坐标"（用于把该面的彩色片几何变换过来）；
- 它的本地朝向四元数。

### 数据结构（草案）

```ts
type SlotKind = "center" | "edge" | "corner";

type Slot = {
  id: number;
  kind: SlotKind;
  center: Vector3;
  quaternion: Quaternion;
  faces: number[];           // 所属面的 index（1/2/3 个）
  faceLocals: Vector3[];     // 在每个所属面坐标系下的局部位置
};

type Piece = {
  id: string;
  kind: SlotKind;
  slot: number;              // 当前所占槽位的 id
  orientation: Quaternion;   // 该 piece 自身相对于槽位的朝向（动画结束后烘焙）
  stickers: { color: string; faceSlotIndex: number }[];
};
```

### F 操作的算法

1. **找正前面**：取相机方向的反向单位向量 `cameraDir`，与 12 个 face normal 点乘，最大者即为"front face"`f`。
2. **筛选受影响块**：所有 `slot.faces.includes(f)` 的槽位，恰好 1 + 5 + 5 = 11 块。
3. **临时编组**：把这 11 块 reparent 到一个临时 `<group>`，绕 face f 的 normal 轴动画旋转到 ±72°。
4. **动画结束烘焙**：
   - 用旋转矩阵把每块的"目标 slot"算出来（按 face f 的 5 重旋转置换表查表）；
   - 把 piece.orientation 乘上 72° 旋转；
   - 把 piece reparent 回根 group，position/quaternion 用新 slot 重新设置。

### 旋转置换表

针对 12 个面，每个面预先算出一组"旋转 72° 后槽位 i → 槽位 j"的置换映射。生成方式：对面 f 的 11 个邻接槽位逐个计算 `R_f(rotate72) * slot.center`，找出与原 slot 列表中最近的那一个即为新位置。

### UI

- 一个浮在 `Canvas` 上的 **F / F' 按钮组**（保留之前单面预览那个交互形式）。
- 一个 **朝向提示条**：右上角显示一个小色块 + 文字 "正前方：黄色"，颜色跟随相机投影实时刷新。
- 动画期间禁用按钮 + 暂停 front face 检测。

## 实施分阶段

每阶段落地后让用户确认效果，发现问题可单独回退。

1. **Step 1 — Piece 渲染骨架**
   - 写槽位表 + Piece 类型 + 初始状态生成。
   - 用 Piece 渲染整个魔方，目标是视觉上和当前一致。
   - 此时 F 操作还没接，纯静态。

2. **Step 2 — 朝向提示上线**
   - 实时检测 front face，渲染右上角提示条。
   - 此时无旋转逻辑，先验证检测稳定。

3. **Step 3 — F 操作（顺时针）**
   - 实现 reparent + 旋转动画 + 槽位置换烘焙。
   - 加 F 按钮。

4. **Step 4 — 扩展**
   - 加 F'（逆时针）。
   - 防抖（动画期间忽略点击）。
   - 视情况加 F2 / 撤销。

## 风险与注意

- **槽位表**最考验细心：60 个棱端点 / 20 个顶点的"所属面"列表必须正确，否则 F 后块会跑错地方。生成方式建议程序化（用面法向 + 顶点投影筛选），不要手填。
- **旋转置换表**也用程序化生成（按"旋转后位置最接近"匹配）。
- 角块/棱块的彩色片需要从"该块所属各面的本地坐标系"变换到"该块自身坐标系"，这一步几何要严谨，否则视觉上彩色片会偏移到块的内部或外部。
- 浮点容差：槽位匹配、面法向投影、相机检测都需要 epsilon。
